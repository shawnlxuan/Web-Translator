# AI 网页翻译浏览器扩展 — 实施计划

## 项目背景

开发一个类似"沉浸式翻译"的浏览器扩展，用于网页 AI 翻译。核心区别在于**上下文感知翻译**：自动收集网页上下文信息（标题层级、周围文本、页面元数据）提供给大模型，从而实现每个句子的精准翻译。支持两种显示模式：对照翻译和仅显示翻译。

| 层面 | 选择 | 原因 |
|---|---|---|
| 扩展框架 | **WXT** | Vite 原生支持，一流的 Chrome/Edge/Firefox 兼容，HMR 热更新 |
| UI | **React 19 + Tailwind CSS 4** | Popup 和 Options 页面使用，最佳 TypeScript 支持 |
| 语言 | **TypeScript 5.8+** (严格模式) | 全类型安全 |
| 句子分割 | `Intl.Segmenter` (浏览器原生) | 现代浏览器零依赖 |
| 包管理 | **pnpm** | 快速、稳定 |
| 测试 | **Vitest** (单元) + **Playwright** (E2E) | Vite 原生 + 跨浏览器 |

## 上下文感知翻译系统（核心差异化功能）

对每个句子，上下文引擎收集结构化的 `SegmentContext`：

1. **文本类型** — 段落、标题、按钮、链接、列表项、表格单元格等
2. **页面元数据** — `<title>`、`<meta name="description">`、`<html lang="...">`
3. **标题层级** — 从 H1→H2→H3 到当前文本的面包屑路径
4. **周围句子** — 前 N 句 + 后 N 句（可配置，默认各3句）
5. **兄弟元素上下文** — 同一容器中相邻元素的文本

这些上下文会结构化为 LLM 提示词：

```
SYSTEM: 你是一名专业翻译，将文本从 {源语言} 翻译为 {目标语言}。规则：保持语气语域，
  对 UI 元素使用简洁自然的译文，不翻译代码/URL，只返回译文。
USER: 翻译: "{句子}"
  所在章节: H1 > H2 > H3
  [上文]: 前几句（仅供理解上下文，不需翻译）
  >>> "{句子}" <<< [这句是待翻译文本]
  [下文]: 后几句
  页面标题: "..."
```

**边界情况处理**：跳过 `<script>`、`<style>`、`<code>`、`<pre>`、`<nav>`、`<footer>`、
SVG、不可见节点。智能检测代码块、导航菜单、已翻译内容。

## 数据流

```
用户点击 Popup → Popup 发送 START_TRANSLATION → Background 转发 → Content Script:
  1. TextExtractor (TreeWalker) → 收集所有可见文本节点
  2. SegmentBuilder → 按块级元素分组，分配段落 ID
  3. SentenceSplitter → 将每段文本分割为句子
  4. ContextCollector → 为每个句子收集上下文
  5. 将 {segments, contexts} 发送至 → Background

Background:
  6. CacheManager 检查 → 缓存命中直接返回
  7. BatchManager → 将缓存未命中项分组（每批5句）
  8. ProviderFactory → 实例化配置的 LLM 提供商
  9. 流式 API 调用 → SSEParser 增量解析
  10. 结果缓存 + 逐句流式回传 Content Script

Content Script:
  11. DisplayManager → 注入翻译（对照模式或替换模式）
  12. MutationObserver → 监听新增 DOM 内容，重复流水线
```

## 两种显示模式

- **对照翻译模式**：原文包裹在 `<span class="tr-original">` 中，译文在
  `<span class="tr-translation">` 中，使用靛蓝色强调、箭头前缀。块级元素使用
  垂直堆叠 + 左侧边框强调样式。
- **仅显示翻译模式**：`textNode.textContent = translation`。原文存储在
  `data-tr-original` 属性中，支持随时切换回对照模式。
- 运行时可切换：通过 Popup 或键盘快捷键 `Alt+M`。

## LLM API 支持

- **OpenAI**：`/v1/chat/completions` 流式，默认 gpt-4o
- **Anthropic**：`/v1/messages` 流式，默认 claude-sonnet-4-20250514
- **自定义**：任何 OpenAI 兼容端点（vLLM、Ollama、OpenRouter、Groq、Azure 等）
- 统一通过 `LLMProvider` 接口，使用 `translateBatchStream()` 方法
- API 密钥存储在 `chrome.storage.local`（不同步到云端）

## 验证方法

1. **手动测试**：在 Chrome 中加载未打包扩展，打开英文维基百科页面，配置 API 密钥后
   点击翻译，验证对照模式效果
2. **模式切换**：翻译过程中切换对照/替换模式，验证所有已翻译段落正确切换
3. **上下文准确性**：用需要上下文才能正确翻译的歧义句子测试（如"它太棒了"在不同
   主题下的翻译），验证是否使用了页面上下文
4. **动态内容**：访问无限滚动页面（如 Twitter），验证新加载内容自动翻译
5. **错误处理**：使用无效 API 密钥，验证 Popup 中的错误提示；触发限流，验证重试行为
6. **缓存命中**：再次翻译同一页面，验证第二次使用缓存（即时显示）
7. **单元测试**：`pnpm test` 全部通过
8. **E2E 测试**：`pnpm test:e2e` 使用模拟 LLM API 服务器全部通过
